Greedy Algorithms — Basic to Advanced

Complete reference with proof strategies, patterns, templates, and curated problems.

1. Core Concepts & Proof Techniques

What is Greedy?

A greedy algorithm makes the locally optimal choice at each step, hoping that these local choices lead to a globally optimal solution. Unlike DP, greedy never reconsiders past decisions.

When Does Greedy Work?

Two conditions must hold:
  1. Greedy choice property — A globally optimal solution can always include the greedy (locally optimal) choice.
  2. Optimal substructure — After making the greedy choice, the remaining subproblem has the same structure.

Proof Techniques for Greedy Correctness

Exchange Argument

Show that swapping any non-greedy choice with the greedy choice doesn't worsen the solution. Most common proof technique.

Greedy Stays Ahead

Show by induction that after each step, greedy's partial solution is at least as good as any other algorithm's.

Structural / Contradiction

Assume greedy isn't optimal, then derive a contradiction with the problem's constraints.

Greedy Thinking Template

4-Step Greedy Recipe:
  1. Sort / Order — What ordering of the input reveals the optimal greedy choice?
  2. Choose — At each step, pick the "best" remaining element.
  3. Validate — Check if the choice is feasible (doesn't violate constraints).
  4. Prove — Use exchange argument or "stays ahead" to convince yourself it's correct.

2. Sorting + Greedy

Pattern: Sort by some criterion, then scan linearly making greedy choices

Assign Cookies

python# Sort children by greed, cookies by size. Match smallest first.
def find_content_children(children, cookies):
    children.sort()
    cookies.sort()
    i = j = 0
    while i < len(children) and j < len(cookies):
        if cookies[j] >= children[i]:
            i += 1  # child satisfied
        j += 1
    return i

Minimum Number of Arrows to Burst Balloons

python# Sort by end position. Shoot at the end of current balloon.
def find_min_arrows(points):
    points.sort(key=lambda x: x[1])
    arrows = 1
    end = points[0][1]
    for s, e in points[1:]:
        if s > end:  # new arrow needed
            arrows += 1
            end = e
    return arrows

Boats to Save People

python# Sort people. Pair heaviest with lightest if possible.
def num_rescue_boats(people, limit):
    people.sort()
    lo, hi = 0, len(people) - 1
    boats = 0
    while lo <= hi:
        if people[lo] + people[hi] <= limit:
            lo += 1
        hi -= 1
        boats += 1
    return boats

3. Interval Scheduling

Pattern: Sort intervals, greedily pick non-overlapping ones

Maximum Non-Overlapping Intervals

python# Classic: sort by end time, always pick earliest-ending interval
def max_non_overlapping(intervals):
    intervals.sort(key=lambda x: x[1])
    count = 0
    end = -float('inf')
    for s, e in intervals:
        if s >= end:
            count += 1
            end = e
    return count

Minimum Removals to Make Non-Overlapping

python# Erase minimum intervals → keep maximum non-overlapping
def erase_overlap_intervals(intervals):
    intervals.sort(key=lambda x: x[1])
    count = 0
    end = -float('inf')
    for s, e in intervals:
        if s < end:
            count += 1  # overlap → remove this one
        else:
            end = e
    return count

Merge Intervals

pythondef merge(intervals):
    intervals.sort()
    merged = [intervals[0]]
    for s, e in intervals[1:]:
        if s <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], e)
        else:
            merged.append([s, e])
    return merged

Meeting Rooms II (Min rooms needed)

pythonimport heapq

def min_meeting_rooms(intervals):
    intervals.sort()
    heap = []  # tracks end times of ongoing meetings
    for s, e in intervals:
        if heap and heap[0] <= s:
            heapq.heapreplace(heap, e)
        else:
            heapq.heappush(heap, e)
    return len(heap)

4. Two Pointers / Scanning Greedy

Pattern: Maintain pointers or a running state, make greedy decisions in a single pass

Jump Game

python# Track farthest reachable position
def can_jump(nums):
    farthest = 0
    for i in range(len(nums)):
        if i > farthest:
            return False
        farthest = max(farthest, i + nums[i])
    return True

Jump Game II (Min jumps)

python# BFS-style: track current window end and farthest reachable
def jump(nums):
    jumps = cur_end = farthest = 0
    for i in range(len(nums) - 1):
        farthest = max(farthest, i + nums[i])
        if i == cur_end:
            jumps += 1
            cur_end = farthest
    return jumps

Container With Most Water

python# Two pointers from both ends. Move the shorter side inward.
def max_area(height):
    lo, hi = 0, len(height) - 1
    result = 0
    while lo < hi:
        area = min(height[lo], height[hi]) * (hi - lo)
        result = max(result, area)
        if height[lo] < height[hi]:
            lo += 1
        else:
            hi -= 1
    return result

Trapping Rain Water

python# Two pointers: track left_max and right_max
def trap(height):
    lo, hi = 0, len(height) - 1
    left_max = right_max = water = 0
    while lo < hi:
        if height[lo] < height[hi]:
            left_max = max(left_max, height[lo])
            water += left_max - height[lo]
            lo += 1
        else:
            right_max = max(right_max, height[hi])
            water += right_max - height[hi]
            hi -= 1
    return water

Gas Station

python# If total gas >= total cost, answer exists. Track min surplus point.
def can_complete_circuit(gas, cost):
    if sum(gas) < sum(cost):
        return -1
    start = tank = 0
    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        if tank < 0:
            start = i + 1
            tank = 0
    return start

5. Heap-Based Greedy

Pattern: Use a heap to always pick the best available element

Task Scheduler

pythonfrom collections import Counter
import heapq

def least_interval(tasks, n):
    counts = list(Counter(tasks).values())
    max_count = max(counts)
    max_count_tasks = counts.count(max_count)
    # Greedy formula: fill frames of size (n+1)
    return max(len(tasks), (max_count - 1) * (n + 1) + max_count_tasks)

Reorganize String (No two adjacent same)

pythondef reorganize_string(s):
    counts = Counter(s)
    heap = [(-cnt, ch) for ch, cnt in counts.items()]
    heapq.heapify(heap)
    result = []
    prev = (0, '')

    while heap:
        cnt, ch = heapq.heappop(heap)
        result.append(ch)
        if prev[0] < 0:
            heapq.heappush(heap, prev)
        prev = (cnt + 1, ch)

    return ''.join(result) if len(result) == len(s) else ""

Minimum Cost to Hire K Workers

pythondef mincost_to_hire(quality, wage, k):
    # Sort by wage/quality ratio. Greedily pick k cheapest qualities.
    workers = sorted(zip(quality, wage), key=lambda w: w[1] / w[0])
    heap = []
    quality_sum = 0
    result = float('inf')

    for q, w in workers:
        heapq.heappush(heap, -q)
        quality_sum += q
        if len(heap) > k:
            quality_sum += heapq.heappop(heap)  # remove largest
        if len(heap) == k:
            result = min(result, quality_sum * w / q)

    return result

6. Greedy on Arrays

Best Time to Buy and Sell Stock II

python# Collect every upward slope
def max_profit(prices):
    return sum(max(0, prices[i+1] - prices[i]) for i in range(len(prices)-1))

Maximum Sum Circular Subarray

python# Answer = max(normal Kadane, total_sum - min_subarray)
def max_subarray_circular(nums):
    max_sum = cur_max = -float('inf')
    min_sum = cur_min = float('inf')
    total = 0
    for num in nums:
        cur_max = max(num, cur_max + num)
        max_sum = max(max_sum, cur_max)
        cur_min = min(num, cur_min + num)
        min_sum = min(min_sum, cur_min)
        total += num
    if total == min_sum:  # all negative
        return max_sum
    return max(max_sum, total - min_sum)

Wiggle Subsequence

python# Count direction changes — greedy on peaks and valleys
def wiggle_max_length(nums):
    if len(nums) < 2:
        return len(nums)
    up = down = 1
    for i in range(1, len(nums)):
        if nums[i] > nums[i-1]:
            up = down + 1
        elif nums[i] < nums[i-1]:
            down = up + 1
    return max(up, down)

7. Greedy on Strings

Remove K Digits (Smallest number)

python# Monotonic increasing stack. Remove peaks (larger digits before smaller).
def remove_k_digits(num, k):
    stack = []
    for d in num:
        while k and stack and stack[-1] > d:
            stack.pop()
            k -= 1
        stack.append(d)
    stack = stack[:len(stack) - k]  # remove remaining from end
    return ''.join(stack).lstrip('0') or '0'

Remove Duplicate Letters (Smallest lexicographic)

pythondef remove_duplicate_letters(s):
    last = {c: i for i, c in enumerate(s)}
    stack = []
    seen = set()
    for i, c in enumerate(s):
        if c in seen:
            continue
        while stack and c < stack[-1] and last[stack[-1]] > i:
            seen.discard(stack.pop())
        stack.append(c)
        seen.add(c)
    return ''.join(stack)

Smallest Subsequence of Distinct Characters

Same as Remove Duplicate Letters — identical algorithm, different LeetCode number (LC 1081 = LC 316).

Minimum Spanning Tree — Kruskal's

python# Greedy: always pick the lightest edge that doesn't form a cycle
def kruskal(n, edges):
    edges.sort(key=lambda x: x[2])
    parent = list(range(n))

    def find(x):
        while parent[x] != x:
            parent[x] = parent[parent[x]]
            x = parent[x]
        return x

    cost = 0
    for u, v, w in edges:
        ru, rv = find(u), find(v)
        if ru != rv:
            parent[ru] = rv
            cost += w
    return cost

Min Cost to Connect All Points

python# Prim's with heap: always extend to the nearest unvisited point
def min_cost_connect(points):
    n = len(points)
    visited = [False] * n
    heap = [(0, 0)]
    cost = edges = 0
    while edges < n:
        w, u = heapq.heappop(heap)
        if visited[u]:
            continue
        visited[u] = True
        cost += w
        edges += 1
        for v in range(n):
            if not visited[v]:
                d = abs(points[u][0]-points[v][0]) + abs(points[u][1]-points[v][1])
                heapq.heappush(heap, (d, v))
    return cost

9. Huffman Encoding & Optimal Merge

Huffman Coding (Greedy Optimal Prefix Code)

pythonimport heapq

def huffman_cost(frequencies):
    # Greedy: always merge the two smallest frequencies
    heap = frequencies[:]
    heapq.heapify(heap)
    total_cost = 0
    while len(heap) > 1:
        a = heapq.heappop(heap)
        b = heapq.heappop(heap)
        merged = a + b
        total_cost += merged
        heapq.heappush(heap, merged)
    return total_cost

Minimum Cost to Merge Stones

Generalization: Merging K piles at a time instead of 2. When K > 2, pure greedy fails — requires interval DP. But the Huffman (K=2) case is a clean greedy.

Connect Sticks / Minimum Cost to Connect Ropes

python# Identical structure to Huffman cost
def connect_sticks(sticks):
    heapq.heapify(sticks)
    cost = 0
    while len(sticks) > 1:
        a = heapq.heappop(sticks)
        b = heapq.heappop(sticks)
        cost += a + b
        heapq.heappush(sticks, a + b)
    return cost

10. Exchange Argument (Advanced Proof Technique)

The Technique

To prove a greedy ordering is optimal:

  1. Take any optimal solution that doesn't use the greedy order.
  2. Find two adjacent elements that are "out of greedy order."
  3. Show that swapping them produces a solution that's at least as good.
  4. By repeatedly swapping, you can transform any OPT into the greedy solution.

Classic Example: Job Scheduling to Minimize Lateness

python# n jobs with processing time p[i] and deadline d[i].
# Schedule to minimize maximum lateness.
# Greedy: sort by deadline (Earliest Deadline First, EDF).
# Proof: swap any out-of-order pair → max lateness doesn't increase.

def min_lateness(jobs):
    # jobs = [(processing_time, deadline), ...]
    jobs.sort(key=lambda x: x[1])  # sort by deadline
    time = max_late = 0
    for p, d in jobs:
        time += p
        max_late = max(max_late, time - d)
    return max_late

Exchange Argument for Custom Sorts

python# Largest Number: prove a+b > b+a ordering is optimal via exchange
from functools import cmp_to_key

def largest_number(nums):
    strs = [str(n) for n in nums]
    strs.sort(key=cmp_to_key(lambda a, b: -1 if a+b > b+a else 1))
    return ''.join(strs) if strs[0] != '0' else '0'

11. Matroid Theory & Greedy Optimality

Why Does Greedy Work for Some Problems But Not Others?

A matroid is a combinatorial structure where greedy is guaranteed to find the global optimum. If a problem can be modeled as optimizing a weight function over a matroid, greedy works perfectly.

A matroid (S, I) has:
  1. Hereditary property: If A ∈ I and B ⊆ A, then B ∈ I.
  2. Exchange property: If A, B ∈ I and |A| < |B|, then ∃ x ∈ B\A such that A ∪ {x} ∈ I.

Examples of Matroids in Algorithms

MatroidGround SetIndependent SetsGreedy Algorithm
Graphic matroidEdges of a graphAcyclic edge sets (forests)Kruskal's MST
Uniform matroidAny set SSubsets of size ≤ kTop-k selection
Partition matroidItems in groups≤ 1 item per groupWeighted bipartite matching (special case)
Linear matroidVectorsLinearly independent subsetsGaussian elimination
Non-matroids: 0/1 Knapsack, Coin Change (arbitrary denominations), and Travelling Salesman are NOT matroids — greedy fails on them.

12. Fractional & Relaxation Greedy

Fractional Knapsack

python# Unlike 0/1 Knapsack, we CAN take fractions → greedy works
# Greedy: sort by value/weight ratio, take greedily
def fractional_knapsack(items, capacity):
    # items = [(value, weight), ...]
    items.sort(key=lambda x: x[0] / x[1], reverse=True)
    total = 0
    for v, w in items:
        if capacity >= w:
            total += v
            capacity -= w
        else:
            total += v * (capacity / w)
            break
    return total

When Greedy ≈ Relaxation

In many optimization problems, the greedy solution corresponds to the LP relaxation (linear programming without integrality constraints). If the LP relaxation happens to be integral (matroid, totally unimodular matrix), greedy gives the exact answer.

Activity Selection as LP

The interval scheduling problem has a totally unimodular constraint matrix, which is why the greedy "earliest end time" algorithm gives the optimal integral solution directly.

13. Monotonic Stack / Deque Greedy

Pattern: Maintain a stack/deque that is always monotonically increasing or decreasing

Next Greater Element

pythondef next_greater(nums):
    result = [-1] * len(nums)
    stack = []  # indices, monotonically decreasing values
    for i, num in enumerate(nums):
        while stack and nums[stack[-1]] < num:
            result[stack.pop()] = num
        stack.append(i)
    return result

Largest Rectangle in Histogram

pythondef largest_rectangle(heights):
    stack = []
    max_area = 0
    heights.append(0)  # sentinel
    for i, h in enumerate(heights):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area

Daily Temperatures

pythondef daily_temperatures(temps):
    result = [0] * len(temps)
    stack = []
    for i, t in enumerate(temps):
        while stack and temps[stack[-1]] < t:
            j = stack.pop()
            result[j] = i - j
        stack.append(i)
    return result

14. Greedy + Binary Search

Pattern: Binary search on the answer, greedy to check feasibility

Template: "Can we achieve value X?" → Binary search on X, use a greedy/linear scan to check if X is feasible.

Split Array Largest Sum (Minimize Max Subarray Sum)

pythondef split_array(nums, k):
    def can_split(max_sum):
        # Greedily pack elements into subarrays of sum ≤ max_sum
        parts = 1
        cur = 0
        for num in nums:
            if cur + num > max_sum:
                parts += 1
                cur = num
            else:
                cur += num
        return parts <= k

    lo, hi = max(nums), sum(nums)
    while lo < hi:
        mid = (lo + hi) // 2
        if can_split(mid):
            hi = mid
        else:
            lo = mid + 1
    return lo

Koko Eating Bananas

pythonimport math

def min_eating_speed(piles, h):
    lo, hi = 1, max(piles)
    while lo < hi:
        mid = (lo + hi) // 2
        hours = sum(math.ceil(p / mid) for p in piles)
        if hours <= h:
            hi = mid
        else:
            lo = mid + 1
    return lo

15. Greedy vs DP — When Greedy Fails

Common Traps

Greedy looks right but is WRONG for:
  • 0/1 Knapsack — items: (60, 10), (100, 20), (120, 30), capacity 50. Greedy by ratio picks items 1+2=160, but optimal is 2+3=220.
  • Coin Change (arbitrary denominations) — coins [1, 3, 4], amount 6. Greedy picks 4+1+1=3 coins, but optimal is 3+3=2 coins.
  • Longest Increasing Subsequence — greedy doesn't explore the "take smaller now to enable longer chain" tradeoff.

How to Detect: Greedy or DP?

SignalGreedyDP
Choices are independent?Yes → GreedyNo → DP
Current choice affects future options?No → GreedyYes → DP
Can you prove exchange argument?Yes → GreedyCan't → DP
Problem asks for count / all solutions?Rarely greedyLikely DP
Problem has "at most K" constraint?SometimesUsually DP
Items can be split (fractional)?GreedyN/A
